# blog-fetch

## 目的

抓取技术博客或技术文章正文，去掉明显无关内容，并保留后续摘要所需的基本元信息。

## 适用场景

- 用户给出博客链接，希望抽取正文
- 需要为技术文章摘要准备干净输入
- 需要把技术内容转成适合后续处理的 Markdown 或纯文本

## 输入

- 单篇文章链接
- 或者来源站点列表
- 时间范围，例如 `7d`、`14d`、`30d`
- 可选上下文，例如抓取重点、是否保留图片链接

## 默认来源

当前默认跟踪以下博客站点的最新文章：

- `https://blog.langchain.com`
- `https://karpathy.bearblog.dev/blog/`
- `https://baoyu.io`

## 获取策略

- 优先使用站点提供的 RSS 或其他订阅入口
- 如果没有稳定订阅入口，则回退到首页或博客列表页抓取最新文章链接
- 先获取最新文章列表，再按需抓取单篇正文
- 抓取失败时，不伪造结果，明确标注失败站点和失败原因

## 输出

- 在给定时间范围内的最新文章列表
- 标题
- 来源
- 发布时间
- 文章链接
- 正文
- 可选的 Markdown 版本
- 可用于后续摘要的清洗结果

## 不负责

- 判断文章是否值得推送
- 生成最终简报
- 写入长期记忆
- 创建推送任务

## 默认要求

- 支持基于时间范围筛选最新文章，例如最近 `7` 天
- 尽量保留正文结构
- 尽量移除导航、广告、推荐阅读等噪音
- 抽取失败时明确返回失败原因
- 当用户给出固定站点时，优先从这些站点获取最新文章，而不是做泛化搜索

## 最小实现范围

第一阶段只要求先完成“最新文章发现”能力：

- 输入站点和时间范围
- 返回时间范围内的文章列表
- 暂不强制完成每篇文章正文抽取

## 当前用法

发现最近 `7` 天文章：

```bash
python3 core/blogs.py --window 7d
```

在 Telegram runtime 中，agent 会优先通过 `fetch_recent_blog_posts` 工具调用这段逻辑，而不是手工猜测最新文章。

运行本地测试：

```bash
python3 -m unittest tests.test_blog_posts
```
