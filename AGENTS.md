# 个人助手项目规范

本项目是一个长期运行的个人生活与工作助手。它既响应对话，也基于长期记忆和既有任务提供主动服务。

## 核心规则

- 对话是统一入口。
- 所有用户可见消息默认使用中文。
- 普通对话默认只进入当前上下文，不自动写入长期记忆。
- 长期记忆只在用户明确表达“帮我记住”“记住这个”“以后按这个来”等语义时写入。
- 长期记忆必须允许通过对话修正、覆盖和删除。
- 任务、提醒、订阅和周期性推送都应由用户显式创建。
- 主动推送默认保持克制；能汇总进简报的内容，不单独打断用户。

## 公开仓库安全规则

- 本仓库已对外公开，任何密钥、令牌、Cookie、会话信息、聊天记录、长期记忆、个人数据和运行日志都不得提交到仓库。
- 所有敏感配置只允许通过本地环境变量或未纳入版本控制的本地文件提供，不允许写入源码、测试快照、文档示例或 skill 定义。
- `memories/`、`.env`、`.obsidian/` 和其他本地运行产物都属于私有数据，不进入版本控制。
- 文档、示例和默认配置必须使用占位值，不能出现真实 bot token、API key、chat id 或真实账户信息。
- 对外公开版本默认只支持研究、摘要、提醒和模拟交易，不接入真实券商、不保存真实交易凭证。
- 如果密钥曾在公开渠道出现，应视为泄露并立即轮换，而不是继续复用。
- Telegram bot 默认必须配置明确的 `allowed chat ids` 白名单，不能以“未配置即全开放”的方式启动。
- 主动推送目标必须属于同一白名单，不允许把 push 目标配置到未授权 chat。

## 业务域

- `life`
  - 日常事务、提醒、待办、计划和长期上下文
- `tech`
  - 技术博客、技术文章、工具、工程实践和技术趋势
- `news`
  - 财经、政治、军事、科技相关新闻和公共事件
- `market`
  - 股票观察、研究、提醒、模拟交易和复盘

这些业务域是平级关系。系统不应因为目录结构而让某一域天然优先。

## 边界

- 每天上午 `09:00` 推送最近 `24h` 简报。
- 简报按业务域分别生成。
- 没有真实内容的域不推送。
- 股票相关能力当前只允许研究、提醒和模拟交易，不允许真实交易。
- 所有高影响现实动作都必须经过人工确认。

## 当前实现约束

- 对话主引擎使用 `DeepAgent`。
- 长期记忆使用 `DeepAgent memory`。
- `skills/` 只放固定、可复用、输入输出明确的工作流定义。
- 固定流程的可执行 Python 代码放在 `core/`，不直接耦合到渠道层。
- Telegram 只是交付渠道，不承载业务规则。
- 定时任务和主动推送由外部调度器触发，不由 agent 自发扩张长期任务。

## 目录原则

- `AGENTS.md` 只保留项目规则和行为边界。
- `README.md` 只保留项目入口、运行方式和当前能力。
- `core/` 放运行核心和共享逻辑。
- `channels/` 放渠道接入。
- `skills/` 放 skill 定义。
- `tests/` 放测试。
- 运行时目录如 `memories/` 按需创建，不在仓库中预置空骨架。
